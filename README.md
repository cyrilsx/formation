Think about the test being spec.

## Naming convertion
```
class MyClassShould {

  @Test void do_somthing_interesting() {
  
  }

}

```

Test creation order 

  1- Name the class
  2- Name the method
  3- Define what you are testing (write the then)
  4- Trigger the code (write the when)
  5- Setup (write the given)
